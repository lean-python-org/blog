A Perfectly Reasonable Question

In his recent article Chris Warrick - How to improve Python packaging, or why fourteen tools are at least twelve too many Chris Warrick asks a perfectly reasonable question - Python has fourteen packaging tools - is that twelve too many? The article includes a look at packaging in two other languages, and provides a very useful overview of fourteen tools in the Python packaging ecosystem. PEP 582 – Python local packages directory also gets a plug1 and the article wonders if PDM might be the future - if so, perhaps we should collectively drop the rest where possible and prefer PDM?
